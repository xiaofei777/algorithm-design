## 四、算法设计思路描述

迷宫问题的回溯算法是一种基于深度优先搜索（DFS）的系统性搜索策略。其核心思想是：从起点出发，沿着一条路径不断探索，当发现当前路径无法到达终点时，就回退到上一步，尝试其他方向，直到找到通往终点的路径或遍历完所有可能的路径。

本实验采用的迷宫标记规则与常规方法不同：使用 `maze[][]=-1` 标记不可通行的墙壁，使用 `maze[][]=dep`（dep 为当前深度，从 1 开始）标记当前路径上的节点，使用 `maze[][]=-2` 标记已探索但无法通向终点的死胡同。这种标记方式使得路径信息直接保存在迷宫数组中，便于回溯时恢复状态。

算法的具体执行过程如下：首先将起点标记为 dep=1，然后按照上、下、左、右四个方向依次尝试移动。对于每个方向，如果新位置在迷宫范围内且为可通行的空地（值为 0），则递归调用回溯函数，并将深度 dep+1。如果某个位置的所有方向都无法到达终点，则将该位置标记为 -2（死胡同），然后回退到上一层继续探索其他方向。当到达终点时，记录当前路径长度，并通过全局变量保存最短路径。

为了找到最短路径，算法需要遍历所有可能的路径，并在每次到达终点时比较路径长度，保留最短的方案。虽然回溯法在最坏情况下时间复杂度较高，但它能够系统地搜索解空间，保证找到最优解（如果存在）。此外，通过标记死胡同，算法避免了重复搜索已证明不可行的路径，提高了搜索效率。

### 时间复杂度分析

在最坏情况下，迷宫中的每个可通行位置都可能被访问多次。设迷宫中有 $n$ 个可通行格子，每个位置最多有 4 个方向可选，但由于标记机制（已访问位置不会被重复访问同一条路径），算法的时间复杂度为 $O(4^n)$，即 $O(2^{2n})$。这是因为每个可通行位置在一条路径中最多被访问一次，而路径的组合数在最坏情况下呈指数增长。对于 $rows \times cols$ 的迷宫，最坏时间复杂度为 $O(2^{rows \times cols})$。

### 空间复杂度分析

空间复杂度主要由以下几部分组成：
- 迷宫数组 `maze[MAX_SIZE][MAX_SIZE]`：$O(MAX\_SIZE^2)$，即 $O(rows \times cols)$
- 最优路径数组 `bestPath[MAX_SIZE][MAX_SIZE]`：$O(rows \times cols)$
- 递归调用栈：最深为迷宫可通行格子数，最坏情况下为 $O(rows \times cols)$

因此，总的空间复杂度为 $O(rows \times cols)$。

## 五、算法核心源代码

```c
#include <stdio.h>
#include <stdlib.h>

#define MAX_SIZE 20
#define INF 999999

int maze[MAX_SIZE][MAX_SIZE];       // 迷宫数组，-1表示墙，0表示通路，-2表示死胡同，正数表示路径步数
int bestPath[MAX_SIZE][MAX_SIZE];   // 存储最短路径的数组
int minLen = INF;                   // 记录最短路径长度
int rows, cols;
int startX, startY, endX, endY;

// 四个移动方向：上、下、左、右
int directions[4][2] = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};

// 复制当前迷宫中的路径到bestPath数组
void copyBestPath() {
    for (int i = 0; i < rows; i++) {
        for (int j = 0; j < cols; j++) {
            bestPath[i][j] = (maze[i][j] > 0) ? maze[i][j] : 0;
        }
    }
}

// 递归回溯算法核心函数
// 参数x, y：当前位置坐标；参数dep：当前深度（步数）
// 返回值：是否找到通路（1找到，0未找到）
int backtrack(int x, int y, int dep) {
    // 到达终点，更新最短路径
    if (x == endX && y == endY) {
        maze[x][y] = dep;
        if (dep < minLen) {
            minLen = dep;
            copyBestPath();
        }
        maze[x][y] = 0;  // 恢复状态，继续搜索其他可能路径
        return 1;
    }

    maze[x][y] = dep;  // 标记当前位置为已访问

    int found = 0;
    for (int i = 0; i < 4; i++) {
        int nx = x + directions[i][0];
        int ny = y + directions[i][1];

        // 检查新位置是否在迷宫范围内且为可通行的空地
        if (nx >= 0 && nx < rows && ny >= 0 && ny < cols && maze[nx][ny] == 0) {
            if (backtrack(nx, ny, dep + 1)) {
                found = 1;
            }
        }
    }

    // 回溯：如果四个方向都无法到达终点，标记为死胡同；否则恢复为空地
    maze[x][y] = found ? 0 : -2;
    return found;
}
```

## 七、实验分析与小结

通过本次实验，我深入理解了回溯法的基本思想和实现方法。回溯法是一种系统地搜索问题解空间的方法，它通过深度优先搜索策略，在搜索过程中不断尝试各种可能的选择，当发现当前选择无法达到目标时，就回退到上一步重新选择。

在迷宫问题中，回溯法的核心在于状态的标记和恢复。本实验采用了特殊的标记方式：使用正数标记路径步数，-1标记墙壁，-2标记死胡同。这种标记方式直观地展示了搜索过程，便于分析算法的执行轨迹。通过对比最短路径标记和搜索后的迷宫状态，可以清晰地看到算法探索了哪些分支，哪些路径被判定为死胡同。

实验结果表明，算法成功找到了从起点(0,0)到终点(4,4)的最短路径，路径长度为9步。路径沿着迷宫左侧向下，再沿底部向右直达终点。在搜索过程中，算法将不可行的分支标记为-2（死胡同），这些标记清晰地展示了回溯过程中被剪枝的搜索空间。

回溯法的优点是能够系统地搜索所有可能的解，保证找到最优解（如果存在）。但其缺点是在最坏情况下时间复杂度较高，为指数级。对于大规模迷宫问题，可以考虑结合启发式信息或使用迭代加深等优化策略来提高搜索效率。

通过本次实验，我不仅掌握了回溯递归算法的设计与实现，还加深了对递归思想、状态恢复和剪枝策略的理解，为今后解决更复杂的组合优化问题奠定了基础。
