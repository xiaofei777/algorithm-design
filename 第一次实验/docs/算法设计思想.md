# 汉诺塔问题算法设计思想

## 一、递归算法设计思想

汉诺塔问题是经典的递归应用问题。算法核心思想如下：

将n个盘子从A柱移动到C柱，需要借助B柱作为辅助。可将问题分解为三个步骤：
1. 将n-1个盘子从A柱移动到B柱（借助C柱）
2. 将第n个盘子从A柱移动到C柱
3. 将n-1个盘子从B柱移动到C柱（借助A柱）

递归出口条件是当n=1时，直接将盘子从源柱移到目标柱。这种方法充分体现了分治策略，将大问题分解为结构相同的小问题，通过递归调用逐步求解。时间复杂度为O(2^n)，空间复杂度为O(n)。

## 二、非递归算法设计思想

非递归实现采用栈来模拟递归过程。每个栈帧保存当前问题的参数（n, from, to, aux）以及执行步骤标记。通过栈的后进先出特性，实现与递归相同的执行顺序。主要步骤包括：
- 将初始问题压入栈，标记为步骤0
- 处理步骤0时分解问题，先压入当前问题（标记为步骤1），再压入n-1子问题
- 处理步骤1时输出当前盘子移动，然后压入当前问题（标记为步骤2），再压入另一个n-1子问题

## 三、组合问题设计思想

组合问题采用分治策略递归实现。对于n个自然数取r个数的组合，每个组合可以分为两种情况：
1. 包含第n个数：在前n-1个数中取r-1个数，加上第n个数
2. 不包含第n个数：在前n-1个数中取r个数

通过递归逐步缩小问题规模，直到r=0时输出组合结果。
