## 三、实验内容

### 3.1 问题描述

一辆汽车加满油后可以行驶 n 千米。旅途中有 k 个加油站。若要使沿途的加油次数最少，设计一个有效的算法，指出应在哪些加油站停靠加油。

### 3.2 数据输入

由文件 input.txt 输入加油站数据。第一行有 2 个正整数 n 和 k，表示汽车加满油后可行驶 n km，且旅途中有 k 个加油站。接下来 1 行中，有 k+1 个整数，表示第 k 个加油站与第 k-1 个加油站之间的距离。第 0 个加油站表示出发地，汽车已加满油。第 k+1 个加油站表示目的地。

### 3.3 结果输出

将编程计算出的最少加油次数输出。如果无法到达目的地，则输出 "No Solution"。

### 3.4 输入输出示例

**输入文件示例：**

```
10 6
1 4 5 3 6 8 3
```

**输出示例：**

```
petrol: 3 at 3, 5, 6 petrol station
```

## 四、算法设计思路描述

### 4.1 贪心策略的选择

汽车加油问题是一个经典的贪心算法应用实例。该问题的核心在于：在确保汽车能够顺利到达终点的前提下，使加油次数尽可能少。

贪心策略的核心思想是：**在每个决策点，都做出当前看来最优的选择**。具体到汽车加油问题，就是**每次都尽可能行驶最远的距离，只在不得不加油的时候才加油**。

### 4.2 算法设计思路

1. **可行性判断**：首先检查所有相邻站点之间的距离，如果存在某段距离大于汽车加满油后的最大行驶距离 n，则直接判定为 "No Solution"，因为无论如何都无法跨越这段距离。
2. **贪心决策过程**：汽车从起点出发时油箱是满的（油量为 n）。在行驶过程中，依次经过各个路段：
   - 每行驶一段距离，就从当前油量中减去该段距离。
   - 判断剩余油量是否足够行驶下一段距离。
   - 如果剩余油量不足以支撑下一段路程（即 current\_fuel < distances\[i+1]），则必须在当前加油站加油，将油量重新加满至 n，并记录该加油站编号。
   - 如果剩余油量足够，则继续行驶，不加油。
3. **最优性证明**：该贪心策略的正确性可以通过反证法证明。假设存在一个最优解，在某个加油站选择了加油，但实际上按照贪心策略此处不需要加油（即剩余油量足够到达更远的加油站）。那么在这个解中，将这次不必要的加油去掉，仍然可以顺利到达目的地，且加油次数更少，这与"最优"矛盾。因此，只在不得不加油时才加油的策略，能够保证加油次数最少。

### 4.3 算法复杂度分析

- **时间复杂度**：O(k)，其中 k 为加油站数量。只需遍历一次所有路段即可得到结果。
- **空间复杂度**：O(k)，用于存储各段距离和加油站点记录。

## 五、算法源代码

```c
// 可行性判断：若任一段距离超过最大行驶能力，则无解
for (i = 0; i <= k; i++) {
    if (distances[i] > n) {
        printf("No Solution\n");
        return 0;
    }
}

// 贪心算法核心：初始满油，遍历每一段路程
current_fuel = n;
for (i = 0; i < k; i++) {
    current_fuel -= distances[i];  // 消耗油量行驶当前段
    // 判断剩余油量是否足够行驶下一段
    if (current_fuel < distances[i + 1]) {
        refuel_stations[refuel_count++] = i + 1; // 记录加油站点编号
        current_fuel = n;  // 加满油
    }
}

// 输出结果
printf("petrol: %d", refuel_count);
if (refuel_count > 0) {
    printf(" at ");
    for (i = 0; i < refuel_count; i++) {
        if (i > 0) printf(", ");
        printf("%d", refuel_stations[i]);
    }
    printf(" petrol station");
}
printf("\n");
```

### 5.1 数据类型和变量说明

| 变量名                 | 类型     | 说明              |
| ------------------- | ------ | --------------- |
| n                   | int    | 汽车加满油后可行驶的最大距离  |
| k                   | int    | 加油站数量           |
| distances\[]        | int数组  | 存储各段距离，共k+1段    |
| refuel\_stations\[] | int数组  | 存储需要加油的站点编号     |
| refuel\_count       | int    | 加油次数计数器         |
| current\_fuel       | int    | 当前油箱剩余油量        |
| filename\[]         | char数组 | 输入文件名，支持命令行参数指定 |

### 5.2 程序模块接口功能说明

- **主函数 main**：程序入口，负责文件读取、数据处理和结果输出。
- **文件读取模块**：从指定文件中读取 n、k 和各段距离数据。
- **可行性检查模块**：遍历所有路段，判断是否存在无法跨越的路段。
- **贪心计算模块**：核心算法，遍历路段并决策是否在加油站加油。
- **结果输出模块**：按指定格式输出最少加油次数及加油站点编号。

## 七、实验分析与小结

### 7.1 实验分析

通过本次实验，我深入理解了贪心算法的基本思想和应用场景。汽车加油问题非常适合用贪心策略解决，因为该问题具有**最优子结构**和**贪心选择性质**：

1. **最优子结构**：整个行程的最优加油方案包含了各个子行程的最优方案。
2. **贪心选择性质**：每次在不得不加油时才加油的选择，不会影响到后续行程达到最优。

在测试过程中，我设计了4组测试数据，覆盖了正常情况和边界情况：

- 测试数据1是题目给出的示例，验证了基本功能。
- 测试数据2和3是自己构造的数据，验证了算法在不同参数下的正确性。
- 测试数据4构造了一个无解的情况（某段距离超过最大行驶能力），验证了异常处理逻辑。

### 7.2 实验小结

通过本次实验，我掌握了贪心算法的设计方法和实现技巧。贪心算法的核心在于找到正确的贪心策略，并证明其正确性。在汽车加油问题中，"只在不得不加油时才加油"这一策略看似简单，却能保证全局最优。这启发我在解决实际问题时，要善于分析问题性质，判断是否可以应用贪心策略。

同时，我也认识到贪心算法并不适用于所有优化问题。只有当问题满足贪心选择性质时，贪心算法才能保证得到最优解。对于不满足该性质的问题，可能需要使用动态规划等其他方法。

## 八、思考题（可选）

### 思考题1：区间覆盖问题

**问题：** 设 X1, X2, …, Xn 是实直线上的 n 个点。用固定长度为 k 的闭区间覆盖这 n 个点，至少需要多少个这样的固定长度闭区间？

**贪心策略：**

1. 先将所有点按坐标从小到大排序。
2. 每次选择最左边未被覆盖的点，以其为起点放置一个长度为 k 的闭区间。
3. 重复步骤2，直到所有点都被覆盖。

该策略的正确性在于：每次放置区间时，总是尽可能向右覆盖更多的点，从而为后续覆盖留下更多空间。

### 思考题2：多机调度问题

**问题：** 将 n 个作业分配到 m 台相同的机器上，使得最大完成时间最小。

**贪心策略（最长处理时间优先，LPT）：**

1. 将所有作业按处理时间从大到小排序。
2. 每次将当前最长的作业分配给当前负载最小的机器。
3. 重复步骤2，直到所有作业分配完毕。

该策略虽然不能保证得到最优解，但可以得到一个近似最优解，且实现简单高效。
