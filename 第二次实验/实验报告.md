# 实验二：贪心算法

## 一、实验目的

基本掌握贪心算法的一般设计方法

## 二、实验内容

### 问题描述
【汽车加油问题】一辆汽车加满油后可以行驶 n 千米。旅途中有 k 个加油站。若要使沿途的加油次数最少，设计一个有效的算法，指出应在那些加油站停靠加油。

### 数据输入
可由文件 input.txt 输入加油站数据。第一行有 2 个正整数 n 和 k，表示汽车加满油后可行驶 n km，且旅途中有 k 个加油站。接下来 1 行中，有 k+1 个整数，表示第 k 个加油站与第 k-1 个加油站之间的距离。第 0 个加油站表示出发地，汽车已加满油。第 k+1 个加油站表示目的地。

### 结果输出
将编程计算出的最少加油次数输出。如果无法到达目的地，则输出"No Solution"。

---

## 三、设计思想（贪心算法）

汽车加油问题采用经典的贪心算法策略。贪心算法的核心思想是在每一步选择中都做出当前看来最好的选择，即局部最优选择，从而希望最终得到全局最优解。

针对汽车加油问题，贪心策略设计如下：
1. 汽车从起点出发时已加满油，当前油量为 n 千米
2. 依次经过每个加油站，每经过一段距离就扣除相应的油量
3. **贪心选择**：在当前油量允许的情况下，尽可能行驶到最远的加油站
4. 当剩余油量不足以到达下一个加油站时，才在当前加油站加油（加满），并记录加油次数
5. 首先进行可行性检查：如果存在任意两个相邻加油站之间的距离超过汽车加满油可行驶的最大距离 n，则直接判定无法到达目的地

这种贪心策略的正确性在于：每次加油都选择在能到达的最远加油站，这样可以最大化每次加油后的行驶范围，从而保证总的加油次数最少。该算法的时间复杂度为 O(k)，其中 k 为加油站的数量，算法效率很高。

## 四、源程序代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

#define MAX_STATIONS 1000

int main(int argc, char *argv[]) {
    int n, k;
    int distances[MAX_STATIONS];
    int refuel_stations[MAX_STATIONS];
    int refuel_count = 0;
    int current_fuel;
    int i;
    char filename[100] = "input.txt";

    if (argc > 1) {
        strcpy(filename, argv[1]);
    }

    FILE *fp = fopen(filename, "r");
    if (fp == NULL) {
        printf("无法打开 %s 文件！\n", filename);
        return 1;
    }

    fscanf(fp, "%d %d", &n, &k);
    for (i = 0; i <= k; i++) {
        fscanf(fp, "%d", &distances[i]);
    }
    fclose(fp);

    for (i = 0; i <= k; i++) {
        if (distances[i] > n) {
            printf("No Solution\n");
            return 0;
        }
    }

    current_fuel = n;
    for (i = 0; i < k; i++) {
        current_fuel -= distances[i];
        if (current_fuel < distances[i + 1]) {
            refuel_stations[refuel_count++] = i + 1;
            current_fuel = n;
        }
    }

    printf("petrol: %d", refuel_count);
    if (refuel_count > 0) {
        printf(" at ");
        for (i = 0; i < refuel_count; i++) {
            if (i > 0) {
                printf(", ");
            }
            printf("%d", refuel_stations[i]);
        }
        printf(" petrol station");
    }
    printf("\n");

    return 0;
}
```

## 五、数据结构及模块接口功能说明

### 数据结构

| 变量名 | 类型 | 说明 |
|--------|------|------|
| `n` | int | 汽车加满油可行驶的最大距离（千米） |
| `k` | int | 加油站的数量 |
| `distances[]` | int数组 | 存储各段距离，distances[i] 表示第 i 段的距离 |
| `refuel_stations[]` | int数组 | 存储需要加油的加油站编号 |
| `refuel_count` | int | 加油次数计数器 |
| `current_fuel` | int | 当前剩余油量可行驶的距离 |

### 模块接口功能

1. **文件读取模块**：从 input.txt 读取 n、k 和各段距离数据
2. **可行性检查模块**：检查是否存在无法跨越的路段
3. **贪心算法核心模块**：遍历各加油站，计算最少加油次数和停靠站
4. **结果输出模块**：输出加油次数和停靠的加油站编号，或输出 "No Solution"

## 十、思考题

### 思考题1：区间覆盖问题

**问题描述**：设 X1，X2，…，Xn 是实直线上的 n 个点。用固定长度为 k 的闭区间覆盖这 n 个点，至少需要多少个这样的固定长度闭区间？

**算法设计**：
1. 首先将所有点按坐标从小到大排序
2. 采用贪心策略：每次选择当前未被覆盖的最左边的点，放置一个区间使其右端点刚好覆盖该点（即区间为 [x, x+k]）
3. 标记所有被该区间覆盖的点
4. 重复步骤2-3，直到所有点都被覆盖

**样例分析**：
输入：7 3，点坐标 [1, 2, 3, 4, 5, -2, 6]
排序后：[-2, 1, 2, 3, 4, 5, 6]
- 第一个区间：[-2, 1] → 覆盖 -2, 1
- 第二个区间：[2, 5] → 覆盖 2, 3, 4, 5
- 第三个区间：[6, 9] → 覆盖 6
输出：3

### 思考题2：机器调度问题（负载均衡）

**问题描述**：已知有 n 个独立的作业由 m 台相同的机器进行加工处理。作业 i 所需的处理时间为 Ti。任何作业可以在任何一台机器上处理，但未完成前不允许间断、划分。试用贪心法设计一种调度方案，使得各机器负载均衡，即最大机器完成时间尽可能小。

**算法设计（最长处理时间优先 LPT）**：
1. 将所有作业按处理时间从大到小排序
2. 依次将每个作业分配给当前总负载最小的机器
3. 重复步骤2，直到所有作业分配完毕

**贪心策略说明**：优先处理时间长的作业，可以尽早将大作业分配出去，避免后期出现大作业无法均衡分配的情况，从而达到负载均衡的目标。
